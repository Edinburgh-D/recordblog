# 多人+多AI协作流程改进方案 v1.0

> 基于10个案例分析，在我们现有Coordinator-Dispatcher模式上加3个机制
> 目标：减少返工、提高效率、降低协调者负担

---

## 一、现状诊断

**当前流程**：
```
用户发任务 → Kimi拆解 → 派给Worker → Worker执行 → Kimi汇总 → 交付用户
                    ↑______________________________↓
                           （信息人工搬运，Worker跨session失忆）
```

**当前问题**：
1. Worker跨session失忆——每次新对话都要重新交代背景
2. 任务不分级——轻任务和重任务用同样流程，效率低
3. 无审批节点——GitHub push、对外发布等关键操作无确认
4. 外部AI结果需人工搬运——手机端测完AI，结果要手动整理发回

---

## 二、3个新增机制

### 机制1：Agent Role Card（角色固化）

**解决什么问题**：Worker每次新对话都失忆，需要重新交代任务和背景。

**怎么做**：
1. 给每个Worker写一份固定的Role Card（Markdown文件），包含：
   - 角色定位（负责什么）
   - 擅长领域（用什么技能）
   - 工作规范（保存文件、发消息格式）
   - 常用上下文（项目背景、关键数据）

2. Role Card存到GitHub/Obsidian，每次启动新任务时自动加载

3. Worker收到任务时，先读取Role Card确认角色，再执行

**示例：黄油小熊Role Card**
```markdown
# 黄油小熊 - 角色卡

## 定位
项目执行主力，负责深度研究、文档撰写、数据分析、代码编写

## 擅长
- 行业研究与竞品分析
- Markdown文档撰写
- Python数据处理
- 结构化输出

## 工作规范
- 所有交付物保存为文件，通过消息发送文件路径
- 复杂任务先出大纲，确认后再写完整版
- 引用数据必须标注来源

## 项目背景（自动加载）
- 项目名称：「父母指南」小程序
- 核心功能：父母育儿风格测评工具
- 技术栈：微信小程序 + Taro 3.x
- 题库：28道题，4维度，16种画像
```

**预期效果**：Worker跨session不再失忆，启动速度提升50%

---

### 机制2：任务复杂度分级

**解决什么问题**：轻任务和重任务用同样流程，轻任务也被过度拆解。

**分级标准**：

| 级别 | 判断标准 | 流程 | 示例 |
|------|---------|------|------|
| **[轻]** | 1步完成，无依赖，明确产出 | 直接执行，无需大纲 | "搜5个竞品" / "改个文案" |
| **[中]** | 2-3步，有依赖，需确认方向 | 先出大纲/框架，确认后执行 | "设计心路历程" / "写分析报告" |
| **[重]** | 多步骤，多依赖，需迭代 | 分阶段交付，每阶段确认 | "完整题库设计" / "MVP开发" |

**执行规则**：
- 用户发任务时，Kimi快速判断级别并标注
- [轻]任务：直接派给Worker，一次出结果
- [中]任务：先让Worker出大纲，用户确认后再执行
- [重]任务：拆成阶段，每阶段结束汇报进度

**预期效果**：轻任务响应时间从30分钟降到5分钟

---

### 机制3：Human-in-the-Loop（关键审批）

**解决什么问题**：关键操作无确认，可能导致误操作（如错误推送、误删文件）。

**审批清单**：

| 操作 | 是否需要审批 | 审批人 |
|------|-------------|--------|
| GitHub文件推送/更新 | ✅ | 逻辑青年 |
| 对外发布内容（公众号/小红书） | ✅ | 逻辑青年 |
| 删除/修改项目核心文件 | ✅ | 逻辑青年 |
| 新建任务/分派Worker | ❌ | Kimi自主 |
| 内部文档撰写/修改 | ❌ | Kimi自主 |
| AI预演/测试类任务 | ❌ | Kimi自主 |

**执行规则**：
1. 触发审批清单的操作，Kimi先暂停，发送审批请求
2. 用户回复`/approve`或"确认"后，继续执行
3. 用户回复`/reject`或"取消"，终止操作

**预期效果**：关键操作零误操作

## 三、中长期优化（本月内）

### 机制4：审阅者Agent正式化

**解决什么问题**：复杂产出直接给用户，有时质量不达标需返工。

**怎么做**：
1. 重要交付物（报告、方案、设计文档）先过Kimi审阅
2. 审阅维度：完整性、准确性、一致性、可执行性
3. 审阅不通过，打回Worker修改，附上具体改进点
4. 审阅通过，标记"已审阅"再交付用户

**预期效果**：交付质量提升，返工率降低50%

---

### 机制5：减少人工搬运

**解决什么问题**：手机端测完AI，结果要手动整理发回，Kimi再人工汇总。

**怎么做**：
1. 外部AI（ChatGPT/DeepSeek等）输出用固定格式
2. Kimi解析固定格式，自动提取关键结论
3. 汇总报告自动生成，无需人工复制粘贴

**固定格式示例**：
```
【AI名称】：ChatGPT
【任务】：心路历程预演
【核心结论】：
1. xxx
2. xxx
【独特发现】：xxx
【适用性】：高/中/低
```

**预期效果**：外部AI结果汇总时间从20分钟降到2分钟

## 四、新协作流程图

```
用户发任务
    ↓
Kimi快速评估（10秒）
    ↓
标注复杂度：[轻]/[中]/[重]
    ↓
┌─────────┬─────────┬─────────┐
│  [轻]   │  [中]   │  [重]   │
│ 直接执行 │先出大纲 │分阶段   │
└─────────┴─────────┴─────────┘
    ↓
Worker执行（加载Role Card）
    ↓
┌─────────┬─────────┐
│简单产出  │复杂产出  │
│直接交付  │先审阅    │
└─────────┴─────────┘
    ↓
触发审批清单？ → 是 → 暂停等用户/approve
    ↓ 否
交付用户
```

## 五、试用验证任务

建议用一个小任务验证新流程：

**试用任务**：[轻] "搜3个育儿测评类竞品的小程序，列出它们的名字和核心功能"

**预期流程**：
1. 用户发任务
2. Kimi标注`[轻]`，直接派给Worker
3. Worker加载Role Card（含项目背景）
4. Worker执行，一次出结果
5. 无需审阅，直接交付
6. 总耗时：5-10分钟
