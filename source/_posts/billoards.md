## 🎯 一、项目核心定位再定义

> **这不是一个“工具类App”，而是一个“线下社交+轻竞技+垂直电商”融合的“兴趣社群平台”**。

### ✅ 核心功能模块梳理：

| 模块 | 功能描述 | 价值点 |
|------|----------|--------|
| 1. 线下门店评价 | 用户可评分、打标签（如“灯光好”“服务差”“有空调”） | 建立信任机制，提升门店曝光 |
| 2. 线上约球 | 按时间/地点/水平/偏好匹配玩家 | 解决“找不到人打球”的痛点 |
| 3. 比赛记录与计分插件 | 实时计分、自动存档、可回放、支持“比洞/比杆”模式 | 增强竞技感与专业性，为内容沉淀打基础 |
| 4. 在线商城 | 售卖台球杆、球、架杆、球桌、配件等 | 垂直电商闭环，高毛利、高复购 |
| 5. 社群与内容生态 | 用户发布打球视频、比赛集锦、技巧教学等 | 提升粘性，形成内容飞轮 |

## 📊 二、市场与用户分析：台球圈，真有“潜力”吗？

### 1. 市场规模 vs. 现实渗透率

- 中国台球爱好者约 **1500万**（含业余+专业）。
- 线下台球厅超 **15万家**，集中在一二线城市及地级市。
- **但90%的台球厅仍靠“现金交易+人情关系”运营**，缺乏数字化工具。

👉 意味着：**存在巨大的“数字化缺口”**。

### 2. 用户画像（典型用户）

| 特征 | 描述 |
|------|------|
| 年龄 | 20-40岁，男性为主（70%），也有年轻女性玩家 |
| 场景 | 朋友聚会、商务应酬、下班放松、情侣约会 |
| 习惯 | 约球难、记分乱、找不到好场地、装备选购麻烦 |
| 情绪需求 | 轻竞技、社交、成就感、放松、仪式感 |

👉 他们不是“只打台球”，而是**在打台球中寻找社交连接和生活仪式感**。

## 💰 三、商业模式评估：能赚钱吗？怎么赚？

### 1. 收入来源（三层变现模型）

| 收入来源 | 模式 | 潜力 |
|----------|------|------|
| **1. 门店增值服务（SaaS）** | 向台球厅收取“门店入驻费+流量推广费+计分系统使用费” | ✅ 高粘性，可复用 |
| **2. 电商佣金（B2C）** | 自建商城或接入品牌，赚取30%-50%毛利 | ✅ 高利润，用户信任度高 |
| **3. 会员订阅（VIP）** | 提供“自动记分+比赛存档+专属球桌”等高级功能 | ✅ 增值服务，提升ARPU |
| **4. 广告与品牌合作** | 与球杆品牌、饮料品牌、运动服饰合作推广 | ⚠️ 低频，但可做 |

### ✅ 关键假设：  
若App能覆盖 **10万用户 + 3000家门店**，  
- 门店年费：200元/店 → 60万收入  
- 电商GMV：500万/年 → 佣金150万  
- 会员费：20元/月 × 5000人 = 120万/年  
→ **年营收可达 330万以上**

👉 这已远超大多数“小而美”App，具备**商业化闭环**。

  phone VARCHAR(20),
  nickname VARCHAR(50),
  avatar_url TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 门店表
CREATE TABLE businesses (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  address TEXT NOT NULL,
  lat DECIMAL(10,8),          -- 纬度
  lng DECIMAL(11,8),          -- 经度
  phone VARCHAR(20),
  description TEXT,
  images TEXT[],              -- 图片URL数组
  created_at TIMESTAMP DEFAULT NOW()
);

-- 门店评价表
CREATE TABLE reviews (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  business_id INTEGER REFERENCES businesses(id) ON DELETE CASCADE,
  rating INTEGER CHECK (rating BETWEEN 1 AND 5),
  tags TEXT[],                -- 如 ['灯光好','服务差']
  content TEXT,
  images TEXT[],              -- 评价配图
  created_at TIMESTAMP DEFAULT NOW()
);

-- 约球活动表
CREATE TABLE match_events (
  id SERIAL PRIMARY KEY,
  creator_id INTEGER REFERENCES users(id),
  business_id INTEGER REFERENCES businesses(id),
  start_time TIMESTAMP NOT NULL,
  end_time TIMESTAMP,
  max_players INTEGER DEFAULT 4,
  current_players INTEGER DEFAULT 1,
  status VARCHAR(20) DEFAULT 'pending', -- pending, ongoing, completed
  created_at TIMESTAMP DEFAULT NOW()
);

-- 比赛记录表（核心！）
CREATE TABLE match_records (
  id SERIAL PRIMARY KEY,
  event_id INTEGER REFERENCES match_events(id),
  player_a_id INTEGER REFERENCES users(id),
  player_b_id INTEGER REFERENCES users(id),
  game_mode VARCHAR(10) CHECK (game_mode IN ('match', 'stroke')), -- 比洞/比杆
  rounds JSONB,               -- [{round:1, scoreA:5, scoreB:3, winner:'A'}, ...]
  total_score_a INTEGER DEFAULT 0,
  total_score_b INTEGER DEFAULT 0,
  winner_id INTEGER REFERENCES users(id), -- 胜者
  status VARCHAR(20) DEFAULT 'completed',
  created_at TIMESTAMP DEFAULT NOW()
);

-- 商品表
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  price DECIMAL(10,2) NOT NULL,
  stock INTEGER DEFAULT 0,
  images TEXT[],
  category VARCHAR(20) CHECK (category IN ('cue', 'ball', 'accessory')), -- 杆/球/配件
  created_at TIMESTAMP DEFAULT NOW()
);

-- 订单表（简化版）
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,

  product_id INTEGER REFERENCES products(id),
  quantity INTEGER DEFAULT 1,
  total_price DECIMAL(10,2),
  status VARCHAR(20) DEFAULT 'pending', -- pending, confirmed, shipped, completed
  created_at TIMESTAMP DEFAULT NOW()
);

## 🚀 三、NestJS 后端核心接口示例（`src/controllers/match.controller.ts`）

import { Controller, Post, Body, Get, Param, UseGuards } from '@nestjs/common';
import { JwtAuthGuard } from '../auth/jwt-auth.guard';
import { MatchService } from './match.service';
import { CreateMatchEventDto } from './dto/create-match-event.dto';
import { CreateMatchRecordDto } from './dto/create-match-record.dto';

@Controller('match')
export class MatchController {
  constructor(private readonly matchService: MatchService) {}

  @UseGuards(JwtAuthGuard)
  @Post('create')
  createEvent(@Body() dto: CreateMatchEventDto) {
    return this.matchService.createEvent(dto);
  }

  @UseGuards(JwtAuthGuard)
  @Post('record')
  recordMatch(@Body() dto: CreateMatchRecordDto) {
    return this.matchService.recordMatch(dto);
  }

  @Get('history/:userId')
  getHistory(@Param('userId') userId: string) {
    return this.matchService.getUserMatchHistory(userId);
  }
}

## 📱 四、Taro 前端页面结构示例（`src/pages/match/index.tsx`）

import Taro, { useState, useEffect } from '@tarojs/taro';
import { View, Text, Button } from '@tarojs/components';
import './index.scss';

const MatchPage = () => {
  const [scoreA, setScoreA] = useState(0);
  const [scoreB, setScoreB] = useState(0);
  const [round, setRound] = useState(1);
  const [record, setRecord] = useState<Array<any>>([]);

  const addPointA = () => {
    const newScoreA = scoreA + 1;
    setScoreA(newScoreA);
    setRecord([...record, { round, scoreA: newScoreA, scoreB, winner: null }]);
  };

  const addPointB = () => {
    const newScoreB = scoreB + 1;
    setScoreB(newScoreB);
    setRecord([...record, { round, scoreA, scoreB: newScoreB, winner: null }]);
  };

  const endRound = () => {
    const winner = scoreA > scoreB ? 'A' : scoreB > scoreA ? 'B' : null;
    setRecord([...record, { round, scoreA, scoreB, winner }]);
    setRound(round + 1);
  };

  return (
    <View className="match-page">
      <Text>第 {round} 局</Text>
      <View className="score-board">
        <Text>选手A: {scoreA}</Text>
        <Text>VS</Text>
        <Text>选手B: {scoreB}</Text>
      </View>
      <Button onClick={addPointA}>A +1</Button>
      <Button onClick={addPointB}>B +1</Button>
      <Button onClick={endRound}>结束本局</Button>

      <View className="history">
        <Text>历史记录：</Text>
        {record.map((r, i) => (
          <View key={i}>
            {r.round}局: {r.scoreA}:{r.scoreB} {r.winner && `🏆${r.winner}`}
          </View>
        ))}
      </View>
    </View>
  );
};

export default MatchPage;
